Getting Started

Welcome to the Zarch ecosystem! This guide will help you install GPM and start using Goscript packages.

Installation

shell curl -sL https://zenv-hub.onrender.com/install.sh | sh

This script installs GPM (Goscript Package Manager) and sets up your environment.

Verify Installation

shell gpm --version
gpm doctor
💡 Tip: GPM stores its configuration in ~/.gpmrc. Packages are installed globally in /usr/local/lib/goscript/.

GPM Basics

GPM (Goscript Package Manager) is the official package manager for the Zarch ecosystem.

Initialize a Project

shell gpm init my-project

Creates a Manifest.toml file with your project metadata.

Install Packages

shell # Install a specific package
gpm install http

# Install with version
gpm install http@1.2.0

# Install all dependencies from Manifest.toml
gpm install

Other Commands

  • gpm uninstall <name> - Remove a package
  • gpm update <name> - Update to latest version
  • gpm list - List installed packages
  • gpm search <query> - Search the registry
  • gpm publish - Publish your package
  • gpm build - Build a .tar.bool archive
  • gpm run <script> - Execute a script

Manifest.toml

The Manifest.toml file defines your package metadata and dependencies.

Example

toml # Manifest.toml

[metadata]
name = "my-package"
version = "1.0.0"
release = "r0"
arch = "x86_64"
description = "My awesome package"
author = "username"
license = "MIT"
main = "main.gjs"

[dependencies]
http = "^1.2.0"
fs = "*"

[scripts]
build = "gpm build"
test = "gpm run test.gjs"

Goscript Language

Goscript is a modern scripting language designed for the Zarch ecosystem.

Hello World

goscript fn main() {
println("Hello, Zarch!")
ret 0
}

Variables

goscript lt name = "Goscript" // Immutable
muts counter = 0 // Mutable
cn PI = 3.14159 // Constant

Functions

goscript fn add(a: int, b: int): int {
ret a + b
}

// Async function
async fn fetch_data(url: string) {
ret await f"curl -s '{url}'"
}

Structs & Enums

goscript struct User {
name: string,
age: int
}

enm Status {
Active,
Inactive
}

lt user = new User{ name: "Alice", age: 30 }
println(user.name)
💡 Learn more: Check the __builtin__ module for standard library functions.

Publishing Packages

1. Login

shell gpm login your-username your-password

2. Build

shell gpm build

Creates a .tar.bool archive in the build/ directory.

3. Publish

shell gpm publish

Uploads your package to the Zarch Hub registry.

⚠️ Important: Make sure your Manifest.toml has a valid name, version, and arch before publishing.

API Reference

Authentication

http POST /v5.2/auth/login
POST /v5.2/auth/register

Packages

http GET /v5.2/package/search?q=query
GET /v5.2/package/{name}
POST /v5.2/package/upload/{scope}/{name}
GET /package/download/{scope}/{name}/{version}/{release}/{arch}

Badges

http GET /badge/{label}-{value}-{color}
GET /badge/{label}-{value}-{color}?style=for-the-badge&logo=github

Registry (Zarch Hub)

The Zarch Hub at https://zenv-hub.onrender.com is the official package registry.

Configuration

shell # ~/.gpmrc
registry = "https://zenv-hub.onrender.com"
scope = "public"
arch = "x86_64"

Package Naming

Packages follow this naming convention:

plain {name}-{version}-{release}-{arch}.tar.bool

Example:
http-1.2.0-r0-x86_64.tar.bool
