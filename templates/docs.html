Overview

Zarch Hub provides a RESTful API for managing APKM packages. All endpoints return JSON responses and use standard HTTP status codes. The API is versioned with the /v5.2/ prefix.

Note: All data is stored on GitHub. No local database is used, ensuring transparency and easy backups.

Base URL

https://gsql-badge.onrender.com

All API endpoints are relative to this base URL. For versioned endpoints, use the /v5.2/ prefix.

Authentication

POST /v5.2/auth/register

Create a new user account.

Request Body

{
  "username": "mauricio",
  "email": "mauricio@example.com",
  "password": "Test123!"
}

Response (201)

{
  "success": true,
  "token": "zarch_xxx...",
  "user": {
    "username": "mauricio",
    "role": "user",
    "created_at": "2026-03-06T12:00:00"
  }
}

Example

curl -X POST https://gsql-badge.onrender.com/v5.2/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"mauricio","email":"mauricio@test.com","password":"Test123!"}'
POST /v5.2/auth/login

Authenticate and get an access token.

Request Body

{
  "username": "mauricio",
  "password": "Test123!"
}

Response (200)

{
  "success": true,
  "token": "zarch_xxx...",
  "user": {
    "username": "mauricio",
    "role": "user"
  }
}

Example

curl -X POST https://gsql-badge.onrender.com/v5.2/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"mauricio","password":"Test123!"}'
GET /v5.2/auth/verify

Verify if a token is valid.

Headers

{
  "Authorization": "Bearer YOUR_TOKEN"
}

Response (200)

{
  "valid": true,
  "user": {
    "username": "mauricio",
    "role": "user"
  }
}

Packages

GET /v5.2/package/{name}

Get detailed information about a package.

Parameters

  • name (path) - Package name
  • version (query) - Specific version (optional)

Response (200)

  "package": {
    "name": "apkm",
    "version": "2.0.0",
    "release": "r1",
    "arch": "x86_64",
    "scope": "public",
    "author": "mauricio",
    "sha256": "68c27c1ef9efd96...",
    "size": 95406,
    "downloads": 42,
    "created_at": "2026-03-01T12:00:00"
  },
  "readme": "
# APKM Package Manager...
"
}

Search

GET /v5.2/package/search

Search for packages by name or description.

Query Parameters

  • q - Search query (required)

Response (200)

{
  "results": [
    {
      "name": "apkm",
      "version": "2.0.0",
      "author": "mauricio",
      "downloads": 42,
      "scope": "public"
    },
    {
      "name": "bool",
      "version": "2.1.0",
      "author": "mauricio",
      "downloads": 15,
      "scope": "public"
    }
  ]
}

Example

curl https://gsql-badge.onrender.com/v5.2/package/search?q=apkm

Upload

POST /v5.2/package/upload/{scope}/{name}

Upload a new package (requires authentication).

Authentication required! Include your Bearer token.

Headers

{
  "Authorization": "Bearer YOUR_TOKEN"
}

Form Data

  • file - .tar.bool package file (required)
  • version - Package version (required)
  • release - Release number (default: r0)
  • arch - Architecture (default: x86_64)

Response (200)

{
  "success": true,
  "package": {
    "name": "apkm",
    "version": "2.0.0",
    "scope": "public",
    "author": "mauricio",
    "sha256": "68c27c1ef9efd96...",
    "size": 95406
  }
}

Example

curl -X POST https://gsql-badge.onrender.com/v5.2/package/upload/public/apkm \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@apkm-v2.0.0-r1-x86_64.tar.bool" \
  -F "version=2.0.0" \
  -F "release=r1" \
  -F "arch=x86_64"

Download

GET /package/download/{scope}/{name}/{version}

Download a package file (direct download, no JSON).

Example

curl -L -o apkm.tar.bool \
  https://gsql-badge.onrender.com/package/download/public/apkm/2.0.0

Rate Limiting

The API implements rate limiting to ensure fair usage:

  • 100 requests per minute per IP address
  • Headers included in responses:
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 99
X-RateLimit-Reset: 60

Complete Examples

1. Register, Login, and Upload

# Register
curl -X POST https://gsql-badge.onrender.com/v5.2/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"devuser","email":"dev@example.com","password":"Dev123!"}'

# Login
TOKEN=$(curl -s -X POST https://gsql-badge.onrender.com/v5.2/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"devuser","password":"Dev123!"}' | jq -r '.token')

# Upload a package
curl -X POST https://gsql-badge.onrender.com/v5.2/package/upload/public/myapp \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@myapp-v1.0.0.tar.bool" \
  -F "version=1.0.0"

2. Search and Download

# Search for packages
curl https://gsql-badge.onrender.com/v5.2/package/search?q=myapp

# Get package info
curl https://gsql-badge.onrender.com/v5.2/package/myapp

# Download
curl -L -o myapp.tar.bool \
  https://gsql-badge.onrender.com/package/download/public/myapp/1.0.0
